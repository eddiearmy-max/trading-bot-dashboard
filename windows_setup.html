Phase 0 — Pre-flight on the Mac

Get the Mac side ready before touching Windows

The risk to avoid: two systems trading simultaneously. If both Mac and Windows fire bots, you'll get duplicate orders. Always-on Windows replaces the Mac entirely.

In your trading-bot directory: 
cd ~/projects/trading-bot
git status
git push
If git status shows uncommitted changes, commit them before moving on. Windows clones from GitHub, so anything not on GitHub is lost.
This file lists every Python package the bot needs. Already generated for you. 
cat requirements.txt
git add requirements.txt
git commit -m "Add requirements.txt for cross-platform install"
git push
The .env file is gitignored, so it won't transfer via Git. Open it now and confirm the 3 values are also in your password manager: 
cat ~/projects/trading-bot/.env
You need: ALPACA_KEY, ALPACA_SECRET, ANTHROPIC_API_KEY.
⚠ Do not do this step until you're ready to immediately move to Windows. Once these are stopped, no bot is firing. Then you start setting up Windows. 
for label in spy-ma qqq-rsi iwm-bb dia-trend aapl-claude aapl-moneypress strc-control allocator caffeinate publish; do
  launchctl bootout gui/$(id -u)/com.eddiearmy.tradingbot-$label 2>/dev/null
done
launchctl list | grep tradingbot
Last line should print nothing — confirming all 10 jobs are stopped.

Phase 1 — Install prerequisites on Windows

The three things Windows needs

Each install is a one-time setup. Reboot after Python finishes installing — it makes life easier.

Download from python.org/downloads — pick the latest 3.12.x Windows installer (64-bit).
⚠ During install, check the box "Add Python to PATH" on the first installer screen. Without it, you'll need to type the full path every time.
After install, verify in PowerShell: 
python --version
pip --version
Download from git-scm.com/download/win.
Recommendation: accept all installer defaults. Git Bash gets installed alongside, but for this guide we'll use PowerShell — it has built-in paste support (just Ctrl+V), is always on PATH, and Windows is built around it. You won't need to open Git Bash manually; we'll only invoke bash.exe for one specific script later.
Verify in PowerShell (Win+X → "Windows PowerShell"): 
git --version
Download from code.visualstudio.com. You'll use it to edit .env and review logs without needing to learn vim/nano on Windows.
Your existing Personal Access Token (PAT) works on Windows too. When Git asks for credentials during the first push/pull, paste your PAT as the "password." If you've lost it, generate a new one at github.com/settings/tokens/new with repo scope.
Windows Credential Manager will cache the PAT after first use — no recurring prompts.

Phase 2 — Clone the repo and create the venv

Recreate the project

Open PowerShell for this phase (Win+X → "Windows PowerShell" or just type "PowerShell" in Start menu). No admin needed yet. 

New-Item -ItemType Directory -Force -Path "$HOME\projects" | Out-Null
cd "$HOME\projects"
git clone https://github.com/eddiearmy-max/trading-bot.git
cd trading-bot
$HOME in PowerShell maps to C:\Users\<you>. The clone lands at C:\Users\<you>\projects\trading-bot. 
cd "$HOME\projects"
git clone https://github.com/eddiearmy-max/trading-bot-dashboard.git
The publish script expects trading-bot-dashboard as a sibling directory of trading-bot. 
cd "$HOME\projects\trading-bot"
python -m venv .venv
We won't activate the venv. Activation requires changing PowerShell's execution policy on some systems. Instead, we'll always invoke Python via .\.venv\Scripts\python.exe — slightly more typing, totally reliable. Task Scheduler uses the same pattern, so this matches production. 
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\pip.exe install -r requirements.txt
Takes ~1 minute. Lots of packages.

Phase 3 — Create the .env file with your secrets

The single file Git doesn't transfer

.env is gitignored on purpose — it has your API keys. You'll recreate it manually.

In PowerShell (still in the project directory): 
cd "$HOME\projects\trading-bot"
New-Item -ItemType File -Path .env -Force | Out-Null
code .env
VS Code opens with an empty file. Paste this template: 
ALPACA_KEY=your_paper_key_id_here
ALPACA_SECRET=your_paper_secret_here
ANTHROPIC_API_KEY=your_anthropic_key_here
Use the values from your password manager. No quotes, no spaces around the =. Save (Ctrl+S). 
.\.venv\Scripts\python.exe test_connection.py
Expected: Status: ACTIVE and a positive buying power. If you get an auth error, double-check the keys in .env.
Market needs to be open for this one, or it'll return HOLD with "market closed": 
.\.venv\Scripts\python.exe bot.py --strategy claude_agent --symbol AAPL
Expected: a HOLD/BUY/SELL with reasoning. If you get auth errors mentioning Anthropic, check the ANTHROPIC_API_KEY in .env.

Phase 4 — Schedule everything via Task Scheduler

Windows's launchd equivalent

Task Scheduler runs your bots on a calendar. There are 10 jobs to recreate. We'll use a single PowerShell script to register them all at once — much faster than clicking through the GUI 10 times.

Win+X → "Windows PowerShell (Admin)" or "Terminal (Admin)". Admin rights are needed to register scheduled tasks.
The scheduler needs absolute paths. In PowerShell: 
cd $HOME\projects\trading-bot
$venv = "$PWD\.venv\Scripts\python.exe"
$bot = "$PWD\bot.py"
$projectDir = "$PWD"
Write-Host "venv: $venv"
Write-Host "bot:  $bot"
Write-Host "dir:  $projectDir"
Make sure all three paths print correctly — no missing files.
Run this in the same PowerShell window where you set the variables above. It creates all the tasks needed (5 equity bots, options bot, heartbeat, allocator, dashboard publisher): 
$weekdays = @("Monday","Tuesday","Wednesday","Thursday","Friday")
$settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable -WakeToRun

# Helper: register a task that runs python.exe with given arguments at a given time
function Register-BotTask {
  param($Name, $Time, $PyArgs, $Days = $weekdays)
  $action = New-ScheduledTaskAction -Execute $venv -Argument $PyArgs -WorkingDirectory $projectDir
  $trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek $Days -At $Time
  Register-ScheduledTask -TaskName $Name -Action $action -Trigger $trigger -Settings $settings -Force | Out-Null
}

# Allocator runs daily before any bot fires
Register-BotTask -Name "TradingBot-Allocator"  -Time "8:20am" -PyArgs "$projectDir\allocator.py 5"

# Equity bots (4 rule-based + heartbeat)
Register-BotTask -Name "TradingBot-SPY-MA"     -Time "9:32am" -PyArgs "$bot --live --symbol SPY --strategy ma_crossover"
Register-BotTask -Name "TradingBot-QQQ-RSI"    -Time "9:33am" -PyArgs "$bot --live --symbol QQQ --strategy rsi_divergence"
Register-BotTask -Name "TradingBot-IWM-BB"     -Time "9:34am" -PyArgs "$bot --live --symbol IWM --strategy bb_squeeze"
Register-BotTask -Name "TradingBot-DIA-Trend"  -Time "9:35am" -PyArgs "$bot --live --symbol DIA --strategy trend_health"
Register-BotTask -Name "TradingBot-STRC-Open"  -Time "9:32am" -PyArgs "$bot --live --symbol STRC --strategy strc_control"
Register-BotTask -Name "TradingBot-STRC-Close" -Time "3:55pm" -PyArgs "$bot --live --symbol STRC --strategy strc_control"

# Hourly AAPL bots — one trigger per hour, 9:30 through 15:30 ET (so the bots match the Mac schedule)
foreach ($h in 9..15) {
  $hour12 = if ($h -gt 12) { $h - 12 } elseif ($h -eq 0) { 12 } else { $h }
  $ampm = if ($h -ge 12) { "pm" } else { "am" }
  $timeClaude = "$($hour12):30$ampm"
  $timeMoney  = "$($hour12):36$ampm"
  Register-BotTask -Name "TradingBot-AAPL-Claude-$h"     -Time $timeClaude -PyArgs "$bot --live --symbol AAPL --strategy claude_agent"
  Register-BotTask -Name "TradingBot-AAPL-Moneypress-$h" -Time $timeMoney  -PyArgs "$bot --live --symbol AAPL --strategy moneypress"
}

# Dashboard publisher — bash script, hourly, all days. Runs via Git's bash.exe.
$publishAction = New-ScheduledTaskAction -Execute "bash.exe" -Argument "publish.sh" -WorkingDirectory $projectDir
$publishTrigger = New-ScheduledTaskTrigger -Once -At (Get-Date) -RepetitionInterval (New-TimeSpan -Hours 1)
Register-ScheduledTask -TaskName "TradingBot-Publish" -Action $publishAction -Trigger $publishTrigger -Settings $settings -Force | Out-Null

Get-ScheduledTask -TaskName "TradingBot-*" | Format-Table TaskName, State -AutoSize
Heads up: This is the most complex script in the migration. If it errors, paste the error message to ChatGPT/Claude — the task scheduler PowerShell API can be finicky and the fix is usually one parameter.
The final line lists all created tasks. You should see ~16 tasks (10 hourly AAPL pairs + the other bots).
Win+R → type taskschd.msc → Enter. In the left pane, click Task Scheduler Library. You should see all TradingBot-* tasks listed with status "Ready."
Tip: right-click any task → "Run" to test it immediately, without waiting for the scheduled time.

Phase 5 — Power management (always-on)

Stop the PC from sleeping

The whole point of moving to Windows is "always on." A scheduled task won't fire if the PC is asleep.

In an Administrator PowerShell window: 
powercfg /change standby-timeout-ac 0
powercfg /change hibernate-timeout-ac 0
powercfg /change monitor-timeout-ac 0
That sets sleep, hibernate, and monitor-off to "never" on AC power.
The monitor-timeout-ac 0 line keeps the screen on too. If you want the screen to turn off but the system to stay awake, set monitor-timeout to e.g. 20 minutes: powercfg /change monitor-timeout-ac 20
The Phase 4 PowerShell script already sets this via -WakeToRun. Just confirm: 
Get-ScheduledTask -TaskName "TradingBot-*" | ForEach-Object {
  "$($_.TaskName): WakeToRun=$($_.Settings.WakeToRun)"
}
All tasks should show WakeToRun=True. If any show False, re-run the Phase 4 script (it'll overwrite with the right settings).
If your PC loses power, you probably want it to come back up on its own. This is a BIOS/UEFI setting: reboot → enter BIOS (usually F2/F10/Del at startup) → find "Power" or "AC Power Loss" or "Restore on AC/Power Loss" → set to "Power On" or "Last State."
Optional, but if you're not at the PC physically, this keeps it always-on across power blips.

Phase 6 — Smoke-test the Windows install

Verify everything works before trusting the schedule

In PowerShell: 
cd "$HOME\projects\trading-bot"
$combos = @(
  @("SPY","ma_crossover"),
  @("QQQ","rsi_divergence"),
  @("IWM","bb_squeeze"),
  @("DIA","trend_health"),
  @("AAPL","claude_agent"),
  @("STRC","strc_control")
)
foreach ($c in $combos) {
  $sym = $c[0]; $strat = $c[1]
  Write-Host "=== $sym / $strat ==="
  .\.venv\Scripts\python.exe bot.py --symbol $sym --strategy $strat
}
Each should produce a HOLD (or similar) with no errors. If anything errors, it's almost always a missing dependency or a wrong path in .env. 
.\.venv\Scripts\python.exe allocator.py 5
Expected: a table showing per-bot caps. Writes bot_caps.json.
publish.sh is a bash script — Windows runs it through Git's bash.exe (installed in Phase 1). From PowerShell: 
bash.exe publish.sh
If bash.exe isn't found, give the full path: & "C:\Program Files\Git\bin\bash.exe" publish.sh
Expected: "Published. Live at: ..." — and the public URL refreshes within a minute. Hard-refresh the dashboard in your browser; the timestamp at the top should be current.
Open Task Scheduler (taskschd.msc) → right-click any TradingBot task → Run. Wait ~10 seconds, then re-check the dashboard or the local logs/ directory for the new entry.

Phase 7 — Cleanup and decommission

Make Windows the sole truth

On the Mac, re-run: 
launchctl list | grep tradingbot
Should be empty. If anything still shows, run the bootout loop from Phase 0 again.
The Mac copy is no longer the source of truth. You can leave it as-is for backup, or zip + move it: 
cd ~/projects
mv trading-bot trading-bot-mac-backup-$(date +%Y%m%d)
Pure preference. Both copies sharing the same GitHub remote isn't a problem unless you accidentally run them concurrently.
